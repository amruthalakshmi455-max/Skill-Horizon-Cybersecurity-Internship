# Project 4: Build Your Own Vulnerable Web Application Module

## Overview
This project demonstrates how insecure coding practices in web applications can lead to SQL Injection (SQLi) vulnerabilities. It includes a vulnerable login system and a secure version implemented using prepared statements.

## Vulnerabilities Tested
- SQL Injection (SQLi)

## Environment
- OS: Windows 10
- Web Server: XAMPP (Apache + MySQL)
- Programming Language: PHP
- Database: MySQL (MariaDB)

## Tools Used
- XAMPP
- phpMyAdmin
- Google Chrome

## Key Learning
- Never trust user input
- Dynamic SQL queries can be exploited
- Prepared statements prevent SQL Injection
- Secure coding practices are essential for web applications
